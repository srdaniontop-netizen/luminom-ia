🔧 Actualizar Reglas de Firestore

Para que el panel de admin funcione correctamente

⚠️ Importante: Sin estas reglas, el panel de admin NO podrá cargar datos de usuarios.

1 Abrir Firebase Console

Haz click en el botón de abajo para ir directamente a las reglas de Firestore:

🔗 Ir a Firebase Console → Reglas

2 Copiar las Reglas

Haz click en "Copiar" y pega el contenido en Firebase Console:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Función helper para verificar si es admin function isAdmin() { return request.auth != null && request.auth.token.email == 'admin@luminom.com'; } // Colección de usuarios match /users/{userId} { // Permitir creación durante el registro allow create: if request.auth != null && request.auth.uid == userId; // Usuario puede leer/actualizar sus propios datos allow read, update: if request.auth != null && request.auth.uid == userId; // Admin puede leer y actualizar todos los usuarios allow read, update: if isAdmin(); } // Colección de chats match /chats/{chatId} { // Usuario puede crear/leer/escribir sus propios chats allow read, write: if request.auth != null && resource.data.userId == request.auth.uid; allow create: if request.auth != null && request.resource.data.userId == request.auth.uid; // Admin puede leer todos los chats allow read: if isAdmin(); } } }

3 Publicar las Reglas

En Firebase Console:

  • Borrar las reglas actuales
  • Pegar las reglas copiadas
  • Click en botón "Publicar"
  • Esperar confirmación

4 Verificar

Después de publicar:

  • Esperar 1 minuto
  • Ir al panel de admin: /admin.html
  • Recargar la página (F5)
  • Deberías ver la tabla de usuarios ✅
✅ ¿Todo funcionando?
Si ves la tabla de usuarios en el panel de admin, ¡las reglas están correctas!
❌ ¿Sigue sin funcionar?
1. Verificar que iniciaste sesión con admin@luminom.com
2. Abrir consola del navegador (F12) y buscar errores
3. Verificar que las reglas se publicaron correctamente en Firebase
